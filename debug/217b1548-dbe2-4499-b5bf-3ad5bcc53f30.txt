# Documentation Research Skill

Research official documentation and best practices for the user's tech stack. This skill orchestrates specialized subagents to find authoritative information efficiently.

## Tech Stack Coverage

This skill provides research for:
- **Frontend**: Astro, Vue 3 (Composition API), TypeScript
- **State**: Nanostores, VueUse
- **Backend**: Appwrite (client SDK, functions, auth)
- **Styling**: Tailwind CSS v4
- **Validation**: Zod
- **UI Libraries**: headlessui/vue, floating-ui/vue, shadcn-ui/vue
- **Additional**: See SITEMAP.md for complete list

## Research Workflow

### Phase 1: Cache Review (Always First)
1. Deploy `cache-researcher` subagent
2. Search `outputs/` folder for recent relevant findings
3. Ignore files older than 60 days
4. Return cache status: FOUND (sufficient) or INSUFFICIENT (need research)

### Phase 2: Active Research (Only if Cache Insufficient)
1. Deploy **in parallel**:
   - `mcp-researcher` subagent - Uses Context7, Astro Doc MCP for official docs
   - `best-practice-researcher` subagent - Uses Tavily for community patterns/articles
2. Both subagents write findings to `outputs/` folder:
   - Format: `doc_[topic]_[YYYY-MM-DD].md` (for MCP research)
   - Format: `web_[topic]_[YYYY-MM-DD].md` (for web research)
3. Wait for both to complete

### Phase 3: Synthesis & Handoff
1. Read all relevant files from `outputs/`
2. Synthesize findings into a concise summary for main agent
3. Highlight:
   - Official documentation findings
   - Best practices discovered
   - Recommended implementation approach
   - Key gotchas or version-specific notes
4. Present summary to user

## Instructions for Claude

When this skill is invoked:

1. **Start with cache-researcher**: Always deploy this subagent first. Do not skip cache check.

2. **Evaluate cache results**: If cache-researcher finds recent, relevant documentation, use those findings. Only proceed to Phase 2 if cache is insufficient.

3. **Parallel deployment**: If research needed, deploy both mcp-researcher and best-practice-researcher simultaneously. Do not wait for one to finish before starting the other.

4. **Let subagents work independently**: Each subagent has its own instructions and will handle writing to outputs/. Do not micromanage their process.

5. **Synthesize findings**: After subagents complete, read the output files they created and present a cohesive summary to the user. Include:
   - What was found in cache (if anything)
   - What new research uncovered
   - Practical next steps
   - Links to authoritative sources

6. **Be concise**: The summary should be comprehensive but information-dense. No fluff, no repetition, just actionable insights.
