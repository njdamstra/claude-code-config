│ **Level 3: AIBrainstormingStage (Actual Animation Logic)**
- Receives `:active="props.active"` (boolean from Theater)
- Receives `:anchor-ref="anchorRef"` (HTMLElement ref for positioning)
- Creates local `stageRef` for the stage container
- Creates `playbackActive` ref (internal state with hysteresis)

**Critical Pattern**: Stage uses `watch()` with 250ms debounce timeout to convert parent's `active` prop into internal `playbackActive` state. This prevents animation restart thrashing from minor visibility flicker.

```typescript
watch(() => props.active, (isActive) => {
  if (isActive) {
    clearTimeout(deactivateTimeout)
    playbackActive.value = true
  } else {
    deactivateTimeout = setTimeout(() => {
      playbackActive.value = false
    }, 250)
  }
})
```   │

│ **useAnimation Return Value - Responsive Plugin Access**

The animation object provides access to the responsive plugin's calculated values:

```typescript
const scaled = computed(() => {
  if (!animation.responsive) {
    // Fallback to base values
    return { /* base values */ }
  }
  
  const { scaled: scaledValues, dimensions } = animation.responsive
  
  return {
    scale: dimensions.value.scale,
    ...scaledValues.value  // All baseValues auto-scaled
  }
})
```

**Critical Pattern**: The responsive plugin exposes:
- `animation.responsive.scaled` - Computed ref with all baseValues auto-scaled based on container width
- `animation.responsive.dimensions` - Container dimensions and calculated scale factor

This means child components receive pre-calculated pixel values (e.g., `orbSize: 72` on mobile, `orbSize: 120` at baseline, `orbSize: 240` on large screens) instead of doing their own responsive calculations!   │

│ **Why useAnimation.play() is NOT called**

Looking at the code, I notice that `animation.play()`, `animation.pause()`, or `animation.stop()` are NEVER called! The only animation method used is:

```typescript
if (animation.isPlaying) {
  animation.stop()
}
```

in `onUnmounted()` for cleanup.

**Critical Insight**: In this pattern, useAnimation is being used as a **reactive dimension calculator**, not as an animation playback controller!

The responsive plugin:
1. Watches container width with ResizeObserver
2. Calculates scale factor based on width vs baseline
3. Auto-scales all baseValues proportionally
4. Exposes scaled values via reactive refs
5. Updates automatically when container resizes

The child components (AiOrb, OrbitingLightBulbs, etc.) handle their OWN animation playback using their internal logic or other animation composables. They just receive responsive dimensions from the parent!   │

│ **Architecture Pattern Summary**

The composition follows a 3-layer architecture:

**Layer 1: Container (WorkflowsSection)**
- Responsibility: Visibility detection
- Tool: `useDebouncedVisibility()`
- Output: Boolean `active` prop

**Layer 2: Theater Wrapper (AIBrainstormingTheater)**
- Responsibility: Teleportation + positioning
- Tool: `AnimatedFloatingElement` component
- Output: Positioned container in teleport layer

**Layer 3: Stage Controller (AIBrainstormingStage)**
- Responsibility: Responsive scaling + playback gating
- Tools: 
  - `useAnimation()` with responsive plugin (dimension calculation)
  - `watch()` with hysteresis (playback control)
- Output: Scaled dimension props to children

**Layer 4: Atomic Components (AiOrb, OrbitingLightBulbs, etc.)**
- Responsibility: Individual animations
- Input: Pre-scaled dimensions + active boolean
- Tool: Own animation logic (e.g., CSS animations, WAAPI, gsap)

This is a **separation of concerns** pattern where each layer has a single, focused responsibility!   │

**MODE 1: Plugin-Only Mode (what AIBrainstormingStage does)**
```typescript
const animation = useAnimation(targetRef, {
  responsive: { /* config */ },
  autoplay: false  // No playback!
})

// Access plugin features:
const scaled = computed(() => animation.responsive.scaled.value)

// Never call: animation.play(), animation.pause()
```
Use for: Responsive calculations, CSS var management, dimension tracking

**MODE 2: Playback Mode (traditional usage)**
```typescript
const animation = useAnimation(targetRef, {
  animation: { preset: 'fadeIn' },
  autoplay: true
})

await animation.play()
animation.pause()
animation.stop()
```
Use for: Controlling actual animations

**Critical Rule**: You can MIX modes! Use responsive plugin for dimensions while using timeline/sequence plugins for playback. The plugins are independent and composable!   │
